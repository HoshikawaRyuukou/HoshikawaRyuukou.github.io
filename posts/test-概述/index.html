以下觀點不一定適用於各專案，請因地制宜。

在開始寫 code 之前

  • 測試文化: 底下的人願意寫，上面的人願意給時間。
  • 測試的順位: 從商業價值最高的功能開始。
  • 較低的測試價值
    • 需求尚未明朗又必須交付成果。
    • 取決於經營者對價值的認定，其實跟程式沒多大關係。
  • 不是所有的程式都可以測試，有時候為了測試，程式需要先重構成可以測試的樣子。
